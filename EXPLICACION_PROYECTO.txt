RESUMEN PROYECTO CLOUDSYNC MANAGER

Lo que hice:

1. ESTRUCTURAS DE DATOS (TDAs/)
- Nodo y ListaEnlazada: lista enlazada basica
- ParametroNodo y ListaParametros: reemplaza diccionarios
- ColaPrioridad: cola ordenada por prioridad (1-10)

2. MODELOS (modelos/)
- Recursos: gestiona CPU/RAM/Storage
- CentroDatos: datacenter con lista de VMs
- MaquinaVirtual: VM con lista de Contenedores
- Contenedor: contenedor con CPU% y RAM MB
- Solicitud: solicitud con prioridad
- GestorSolicitudes: administra cola de solicitudes
- Instruccion y EjecutaInstrucciones: ejecuta comandos del XML

3. LECTOR XML (XMLReader/)
- Lee el xml y crea estructura de listas anidadas
- Centro ---> VMs ---> Contenedores (3 niveles)
- Autoejecuta instrucciones al cargar

4. MENUS (main.py)
Opcion 1: Cargar XML
Opcion 2: Gestion Centros 
Opcion 3: Gestion VMs 
Opcion 4: Gestion Contenedores 
Opcion 5: Gestion Solicitudes 

5. FUNCIONALIDADES CLAVE
- Calculos dinamicos: porcentajes CPU/RAM en tiempo real
- Validacion de recursos antes de crear/agregar
- Actualizacion automatica: al agregar/eliminar contenedor actualiza recursos de VM
- Cola de prioridad: solicitudes ordenadas automaticamente
- Migracion de VMs: actualiza contador size en ambos centros
- Instrucciones del XML se ejecutan automaticamente al cargar
- Contenedores usan CPU% y RAM MB (no GB como los centros)
- Estado de contenedores se guarda en objeto (Running, Pausado, etc.)
- Todas las validaciones devuelven (exito, mensaje)
